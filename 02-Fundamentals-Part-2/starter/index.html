32. Activating Strict Mode

  • zum aktivieren muss am Anfang "use strict"; im js-File hinterlegt werden
  • wenn Code davor steht funktioniert es nicht, Kommentare dürfen aber davor
  • macht es einfacher Fehler zu vermeiden
  • lässt JS Fehler anzeigen, die es sonst ignorieren würde

33. Functions

  • ...können immer wieder im Code verwendet werden
  • Standard Deklaration und Aufruf: 
    function logger() {
      console.log("My name is Stefan");
  }
  
  logger();
  • Funktion mit return-Statement: 
            function fruitProcessor(apples, oranges) {
          console.log(apples, oranges);
          const juice = `Juice with ${apples} apples and ${oranges} oranges`
          return juice;
      }
      
      const appleJuice = fruitProcessor(5, 0); // Ohne Speichern in Variable gibt Konsole nur "5 0" aus
      console.log(appleJuice);
      // console.log(fruitProcessor(5, 0)); // Alternativ Funktion gleich in die Ausgabe, ohne Speichern
      
      const appleOrangeJuice = fruitProcessor(2, 4);
      console.log(appleOrangeJuice);
  • werden benutzt um "Dry Code" zu sschreiben, heißt das ähnliche Ausdrücke nicht wieder und wieder einzeln aufgeschrieben werden
  • Parameter sind die Platzhalter in der Funktion und Argumente die Werte, die in die Parameter eingesetzt werden
  • JS behandelt Funktionen als ganz normale Werte daher können sie in Variablen gespeichert werden

34. Function Declaration vs. Expression

  • Declaration: 
            function calcAge1(birthYear) {
          return 2037 - birthYear;
      }
      
      const age1 = calcAge1(1986);
  • Expression is function without a name: 
    const calcAge2 = function (birthYear) {
      return 2037 - birthYear;
  }
  
  const age2 = calcAge2(1986);
  • big difference: you can call declared function before they even have been declared

35. Arrow Functions

  • const calcAge3 = birthYear => 2037 - birthYear;
  • words function and return and parenthesis can be omitted if only one parameter and one line of code
  • can not use for every case because they cannot handle the this keyword
